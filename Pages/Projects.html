Twitter API Project

Project description:

The Twitter API project was an individual project split into 2 parts, with the goal of better understanding API interaction and web requests. The first was the Twitter Bot Guessing Game. This part of the project used the Twitter API, Python libraries, and basic game development structure to develop a simple guessing game where a query is sent to Twitter to obtain popular tweets, those tweets are displayed, and the user inputs which tweet they believe has the most likes.
The second portion of the project involved interacting with the Twitter API on a more in depth level, using specific functionality of the Twitter API to control many elements of the Twitter account. This functionality included: posting tweets to main timeline, getting tweets from specific users and the bot's own timeline, direct messaging users, and querying general data based on inputs.

Relevant skills:

  • Object-oriented development and utilizing 3rd party libraries in Python
  • Interaction with an external API
  • JSON parsing for data collection and filtration
  • Developing a simple application GUI
  • Learning new technologies using online documentation

Natural Language Processing

Project description:

This was a team project completed at a hackathon during my internship at Principal Financial. The goal was to automate the scanning process of real estate memorandums to collect information, saving human effort. The final product consisted of a dynamic map which interfaced with our database built to store the scanned information collected by the Natural Language Processing system.

My role:

I worked on the back-end of the project, using a mix of Python, AWS Lambda, and Amazon Comprehend to develop a scanning function. This function worked by accessing a document from an S3 Bucket, scanning the document using AWS Lambda and Python, then running commands through Amazon Comprehend to retrieve relevant information in the form of a JSON file. The retrieved information was returned to be parsed for database entry.

Relevant skills:

  • Coding in Python, specifically accessing external databases
  • AWS basics, such as S3 buckets, Lambda, and general operation
  • Teamwork, resource allocation and time management

Priority Queue Based Web Crawler

Project description:

The objective for this team assignment was to develop a web crawler with a priority queue, with the purpose of finding the most relevant links on a given Wikipedia page. The crawler would parse through a given number of pages and return links, which would be given a priority value. From there, the links were sorted in a priority queue and output in a particular order such that links deemed most relevant would appear first. The result was an algorithm that could accurately crawl through a webpage, locate relevant Wikipedia links, and return them to the user by priority.

My role:

I developed the priority queue and "tuple" objects, which were used as generic containers for each link. Links were given a priority value based on certain features, and the priority queue used time efficient algorithms to place and reorder the queue based on priority.

Relevant skills:

  • Object oriented programming in Java
  • Optimization of algorithms by reducing O(n) time
  • Teamwork, resource allocation and time management
